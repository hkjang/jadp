JADP 개발 문서

Spring Boot 기반 OpenDataLoader PDF 연계 샘플입니다. 비동기 Job 구조, 동기 미리보기, Swagger 문서, 브라우저 테스트 화면, 파일 다운로드 API를 함께 제공합니다.

Swagger UI OpenAPI JSON 테스트 화면 Upstage 콘솔

주요 엔드포인트

경로 설명
POST /api/v1/pdf/convert 비동기 변환 Job 생성
POST /api/v1/pdf/convert-sync 소형 문서 동기 변환 및 미리보기 반환
GET /api/v1/pdf/jobs/{jobId} Job 상태, 처리시간, 생성 파일 조회
GET /api/v1/pdf/jobs 최근 Job 목록 조회
GET /api/v1/pdf/files/{fileId} 생성 산출물 다운로드 또는 브라우저 미리보기
GET /api/v1/pdf/config/options UI용 옵션 사전 조회
POST /api/v1/pii/detect PDF 구조 기반 PII 탐지 및 bounding box 반환
POST /api/v1/pii/mask PII 탐지 후 masked PDF 또는 이미지 생성
GET /api/v1/pii/files/{fileId} masked 파일 다운로드
POST /v1/pii-masker/oac/detect Upstage PII Masker OAC 호환 응답 구조로 탐지 결과 반환
POST /v1/pii-masker/oac/mask Upstage PII Masker OAC 호환 구조와 masked 파일 링크 반환
GET /upstage Upstage Parse 및 PII OAC 호환 API를 따로 검증하는 전용 테스트 페이지

설계 메모

OpenDataLoader Java Quick Start는 Config 와 OpenDataLoaderPDF.processFile() 중심 예시를 제공합니다. 이 샘플은 버전 간 setter 차이를 줄이기 위해 반사 기반 어댑터를 사용하고, README에 버전 고정과 검증 절차를 함께 명시합니다.

민감정보 옵션은 라이브러리 기본 안전장치에 더해 텍스트 산출물 후처리 치환도 수행하도록 설계되어 있어, 버전 차이로 특정 safety option setter가 달라져도 동작을 최대한 유지합니다.

PII 탐지 API에서 PDF는 OCR을 사용하지 않고 OpenDataLoader JSON 구조와 bounding box를 기반으로 분석합니다. PNG/JPG는 별도 app.vllm-ocr.* 설정이 있는 경우에만 OpenAI 호환 vLLM 비전 API를 호출합니다.

이미지형 PDF 대응은 하이브리드 full 모드 1차 분석 뒤에 표/그림 영역 재시도와 중심 컬럼/밴드 타일 재시도를 추가해, 이미지 안 텍스트의 bbox 회수를 최대한 높이도록 보강했습니다.

masked PDF는 재검색 가능한 원문 텍스트가 남지 않도록 페이지를 렌더링한 뒤 개인정보 영역을 덮어쓴 라스터 PDF로 생성합니다.

Hybrid 운영 메모

공식 Hybrid Mode 문서 기준으로 이 프로젝트는 docling-fast, hybrid_url, hybrid_timeout, hybrid_fallback, hybrid-mode=full 흐름을 앱 설정과 UI에 반영합니다.

복잡한 표나 그림이 많은 PDF는 하이브리드가 유리합니다. 다만 속도는 Java-only 보다 느려질 수 있으므로, 대량 단순 문서는 Java-only 와 비교 운영하는 것이 좋습니다.

그림 설명과 차트 설명을 출력 결과에 실제 포함하려면 하이브리드 서버에서 picture description 을 켜는 것만으로는 부족하고, 클라이언트 요청 기본값도 full 이어야 합니다.

도커 배포용 앱 이미지와 하이브리드 서버 이미지는 루트의 DOCKER.md 와 scripts/docker-run.ps1 기준으로 바로 실행할 수 있게 준비되어 있습니다.
